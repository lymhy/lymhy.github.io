玩客云刷机教程

发表于2024-02-10|更新于2025-03-09|刷机指南
|浏览量:

​
玩客云刷armbian安装1panel，docker，qb等，经验分享。

一，拆机篇
        准备材料
                1.数据线,网线各一条

                2.用于短接的镊子

                3.所需软件

      撕开玩客云后面的塑料双面胶，拧开螺丝即可。

玩客云v1.0和v3.0的短接图

二，刷armbian固件
        接线　　
在不通电的状态下电脑连接玩客云HDMI旁边的usb端口，短接后连接电源，此时玩客云指示灯会熄灭，电脑会弹出提示音。

        刷机　　
　　下载usbburningtools2.1.68(听说高于2.2版本会报错)。操作流程：选择文件→导入固件包→点击开始即可，刷写完成后记得点击停止，以免二次刷写。

　　固件包可以从下面的git仓库下载，一定要选择后面带burn（线刷包）的固件https://github.com/hzyitc/armbian-onecloud

　           burn 线刷包，edge 开发版，current稳定版， mini核心板

　　下载完成后解压为img后缀的文件

　　

　　刷写完成后准备重启玩客云，连接电源,网线到路由器，玩客云指示灯闪烁后稳定，开机成功。可以修改默认的主机名，命令如下：

hostnamectl set-hostname name
tips：name改成你自己想要的主机名即可。

　    登录玩客云
　　使用ssh工具(常见的如putty，xshell等ssh客户端)连接玩客云IP，IP在路由器管理后台即可查到。　

ssh root@玩客云IP

　　首次登录要求更新密码，输入密码后会提示创建账户，不想创建可以ctrl+c跳过，不跳过会进入创建账户，选择bash,时区,语言等。

        检查时区
date-R

　　 非国区需要更改时区： 

 cp/usr/share/zoneinfo/Asia/Shanghai/etc/localtime

        换源
             软件可以正常安装可以不替换。

             打开源编辑列表        

nano/etc/apt/sources.list

            换源是可以把原先的软件源用“#”注释掉，防止误删后悔。

#阿里源

deb https://mirrors.aliyun.com/debian-security/bullseye-security main

#deb-src https://mirrors.aliyun.com/debian-security/bullseye-security main

deb https://mirrors.aliyun.com/debian/bullseye-updates main non-free contrib

#deb-src https://mirrors.aliyun.com/debian/bullseye-updates main non-free contrib

deb https://mirrors.aliyun.com/debian/bullseye-backports main non-free contrib

#deb-src https://mirrors.aliyun.com/debian/bullseye-backports main non-free contrib

#中科大源

deb https://mirrors.aliyun.com/debian/bullseye main non-free contrib

#deb-src https://mirrors.aliyun.com/debian/bullseye main non-free contrib

deb https://mirrors.aliyun.com/debian-security/bullseye-security main

#deb-src https://mirrors.aliyun.com/debian-security/bullseye-security main

deb https://mirrors.aliyun.com/debian/bullseye-updates main non-free contrib

#deb-src https://mirrors.aliyun.com/debian/bullseye-updates main non-free contrib
随便选一个添加即可。

          修改完后按Ctrl+O保存，再按Ctrl+X退出或者先退出，y确认保存。

       更新(非必要)          

apt-getupdate && apt-getupgrade              

　　自此armbian安装完成。

三，应用篇
　　本篇文章一共推荐俩个管理面板，供读者自由选择。

　　1.安装1panel　　
　curl -sSL https://resource.fit2cloud.com/1panel/package/quick_start.sh -o quick_start.sh && bash quick_start.sh
　　2.安装casaos　　
curl -fsSL https://get.casaos.io | sudo 
 　 网络不好的人可以试试这个国内源(使用国内源安装无法自动升级casaos)

wget -qO- https://get.casaos.io | bash
　　3.安装docker
apt install docker.io
　　4.安装qbittorrent
由于玩客云32位处理器，而面板提供的bt软件均为64位版本，我们需要通过APT自行安装。

　　默认账号：admin

　　默认密码：adminadmin

　　执行以下命令　　　

sudo apt install qbittorrent-nox
　　通过nano文本编辑器打开qbittorrent服务文件            

  nano /etc/systemd/system/qbittorrent.service
         输入以下内容（配置信息）后，ctrl+x保存。         　

　 [Unit]

                Description=qBittorrent Daemon Service

                After=network.target

                [Service]

                User=root

                ExecStart=/usr/bin/qbittorrent-nox

                ExecStop=/usr/bin/killall -w qbittorrent-nox

                [Install]

                WantedBy=multi-user.target
         执行以下命令，更新刚刚配置的服务，设置开机启动以及现在启动该服务。         　

　sudo systemctl daemon-reload

sudo systemctl enable qbittorrent.service

  sudo systemctl start qbittorrent.service                   

     5.安装Samba（文件共享）
    执行以下命令，更新软件包列表，并通过APT安装Samba应用。

sudo apt update
sudo apt install samba
查看安装状态

systemctl status nmbd
    执行以下命令，打开Samba的配置文件。

sudo nano /etc/samba/smb.conf
    在文件的末尾添加以下内容（配置信息）后，Ctrl+X保存，以创建共享。

[共享显示的名称，可自定义]

[flash]

#自定义备注信息

comment = flash

#路径，下列是我的共享路径，要设置成你的，怎么查询路径，打开面板,找到文件夹图标打开，左侧位置>下面找到要挂载的U盘，鼠标移到目标文件夹右边的点，点击选择拷贝路径，再粘贴到path

path = /media/devmon/sda1-usb-Netac_OnlyDisk_5/lost+found/

#共享用户名

valid users = root

#只读 yes=是 no=否

read only = no

#来宾

guest ok = no

#创建文件时权限，权限按需设置

create maks = 0777

#创建目录时权限，权限按需设置

directory mask = 0777

#是否可被看见

browseable = yes

available = yes

writable = yes
    上述设置会默认共享root目录，如不想显示root目录，可以增加在[homes]节点中，“comment = Home Directories”，“browseable = no”的下面增加下列配置信息。

#是否可被看见

browseable = no

available = no

writable = no
    创建用户密码，注意用户名是否是第2步骤用户名。

sudo smbpasswd -a root
    执行以下命令，重启Samba服务以应用配置更改。

sudo systemctl restart smbd
    执行以下命令，设置Samba开机自启。

sudo systemctl start smbd
sudo systemctl enable smbd
6.安装Cpolar（内网穿透）
    执行以下代码，安装Cpolar。

curl -L https://www.cpolar.com/static/downloads/install-release-cpolar.sh | sudo bash
    查看版本号，正常显示版本号及安装成功。

cpolar version
    登录cpolar官网后台，点击左侧的验证，查看自己的认证token，之后将token贴在命令行里。

cpolar authtoken xxxxxx
    向系统添加服务。

sudo systemctl enable cpolar
    启动cpolar服务

sudo systemctl start cpolar
    在浏览器上访问本地端口http://玩客云IP:9200,使用cpolar邮箱账号登录Cpolar web UI管理界面，即可开始使用cpolar。

    登录完cpolar本地管理界面后，在“隧道管理”-“创建隧道”中，可以给SSH、CasaOS、qBittorrent等创建隧道（端口）这样就可以实现远程访问了。

参考Cpolar的官方文档：https://www.cpolar.com/docs

7.安装Portainer
直接输入命令

docker volume create portainer_data
docker run -d -p 9000:9000 -p 8000:8000 –name portainer –restart always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:latest
访问玩客云ip+端口9000就可以进入prtainer

8.安装Home Assistant
输入一下命令   

docker pull homeassistant/home-assistant
然后在prtainer里面创建容器，选择刚才拉取的镜像文件。

然后通过玩客云IP+8123端口访问，出现配置界面。

9.常见的默认端口
CasaOS：8181

Cpolar：9200

qBittorrent ：8080

FAST OS DOCKER：8081

青龙：5700

Home Assistant：8123

四，几个常见命令
为了方便对linux系统不熟悉的玩家，笔者在下面写几个常见命令给大家参考。

linux常见的两个查找命令
　　eee.txt为查找的文件名

　　find /-name eee.txt 和 locate eee.txt

　　

linux修改文件夹权限
        sudo chmod 600 ××× （仅所有者读和写的权限）

        sudo chmod 644 ××× （所有者读和写的权限，组用户只有读的权限）

        sudo chmod 700 ××× （仅所有者有读和写以及执行的权限）

        sudo chmod 666 ××× （每个人都有读和写的权限）

        sudo chmod 777 ××× （每个人都有读和写以及执行的权限）

tip:以上方法只能改变单个文件或文件夹

修改文件夹下所有文件的权限

        sudo chmod -R 777 filename(文件名路径)

filename为要修改的文件夹名字。-R应该是表示递归修改filename文件夹下所有文件的权限。

其实整个命令的形式是

        sudo chmod -（代表类型）×××（所有者）×××（组用户）×××（其他用户）

三位数的每一位都表示一个用户类型的权限设置。取值是0～7，即二进制的[000]~[111]。

这个三位的二进制数的每一位分别表示读、写、执行权限。

如000表示三项权限均无，而100表示只读。这样，就有了下面的对应：

        0 [000] 无任何权限

        4 [100] 只读权限

        6 [110] 读写权限

        7 [111] 读写执行权限

查询文件（或文件夹）权限的命令
        ls -l 文件名称 （文件夹将-l改为-ld）

        为所有者（u）增加脚本执行权限(+v)

        chmod u+x install_nginx.sh

常见的APT命令
        apt-get clean - 删除所有已下载的包文件

        apt-get update - 取回更新的软件包列表信息

        apt-get upgrade - 进行一次升级
